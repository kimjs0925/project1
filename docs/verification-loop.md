# 아침대화 검증 루프

아침대화 앱은 수업 중 학생이 직접 쓰는 도구라서 "코드가 실행됨"만으로는 충분하지 않다. 변경 후에는 자동 확인, 독립 점검, 실제 화면 확인을 조합한다.

## 1. 자동 확인

기본 하네스 체크:

```powershell
npm run check
```

이 체크는 다음 불변식을 확인한다.

- 학생 화면 상단 학급 표시가 `5학년 6반`이다.
- `/morning/student`, `/morning/admin` 라우트가 유지된다.
- 아침대화 피드백 API와 로컬 fallback이 유지된다.
- 아침대화 기록 저장 API가 유지된다.
- 주요 secret 파일명이 정적 파일로 노출되지 않도록 방어 코드가 남아 있다.

## 2. 기능 점검

서버 실행:

```powershell
npm start
```

학생 화면에서 확인:

- 학생 번호와 비밀번호 입력 후 대화가 시작된다.
- 질문, 답변 기록, 다음 질문 진행이 이어진다.
- 마이크가 안 될 때 직접 기록으로 답변할 수 있다.
- 카메라 안내 문구가 화면 저장을 암시하지 않는다.
- AI 피드백 실패 시에도 대화가 멈추지 않는다.

관리자 화면에서 확인:

- 관리자 비밀번호로 들어갈 수 있다.
- 오늘 아침대화 목록이 표시된다.
- 학생별 상세와 날짜별 기록을 확인할 수 있다.
- 삭제 시 날짜 불일치 보호가 유지된다.

## 3. 개인정보/안전 점검

`docs/security-privacy-checklist.md`를 보고 다음을 확인한다.

- API 키가 HTML/JS로 직접 들어가지 않았다.
- 학생 대화 전문을 외부 서비스로 새로 보내는 경우 목적과 보존을 설명할 수 있다.
- 표정 신호는 학생 마음을 단정하지 않는 보조 신호로만 표현된다.
- 새 저장 필드는 보존 기간과 삭제 동작을 고려한다.

## 4. 실패 기록

검증 중 놓친 문제가 발견되면 `docs/decisions-log.md`나 이 문서에 불변식으로 승격할 수 있는지 확인한다. 반복되는 문제는 `scripts/harness-check.mjs`에 자동 검사로 추가한다.
