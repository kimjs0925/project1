# 사회정서앱 작업 인수인계

새 채팅에서 사용자가 **"사회정서앱 읽어"**라고 말하면 이 파일을 먼저 읽고 현재 상태를 이어서 작업한다.

## 프로젝트 요약

- 위치: `C:\Users\user\Desktop\앱 개발`
- 앱 성격: 학생/관리자용 갈등 해결 사회정서 활동 앱
- 프론트엔드: 단일 `index.html`
- 백엔드: `server.js` Express 서버
- 기본 저장 키: `localStorage`의 `conflictAppState`
- 민감 파일: `.env`, `openaiapi.env`, `제미나이.env.txt`, `.data/`는 읽거나 노출하지 않는다.

## 현재 주요 기능

- 학생 페이지
  - 학생 번호 선택
  - 갈등 상황 확인
  - 상황별 등장인물 선택
  - 등장인물이 느꼈을 감정 입력
  - 입력 내용은 `state.students[studentId].responses[situationId]`에 저장

- 관리자 페이지
  - 관리자 암호는 `1`
  - 외부 링크 설정
  - 상황 추가/수정/삭제
  - 등장인물과 등장인물별 공감 기준 관리
  - 학생 감정 입력 삭제
    - 삭제할 상황 선택
    - 학생 번호 필터
    - 인물 필터
    - 개별 감정 입력 삭제
    - 선택한 상황의 감정 입력 전체 삭제
  - 감정 워드클라우드
    - 전체/상황별/등장인물별 필터
    - `wordcloud2.js` 실패 시 기본 캔버스 렌더링
  - 학생 공감 결과 표시

## 서버/동기화 상태

- `server.js`에 `/api/state`가 있다.
- 여러 기기 학생 입력을 모으려면 선생님 컴퓨터에서 서버를 켜고 학생들이 같은 서버 주소로 접속해야 한다.
- 학생이 복사한 `index.html`을 여는 경우를 위해 `teacherServerUrl`이 `http://192.168.1.19:3000`으로 설정되어 있다.
- 학생 상태 동기화는 서버가 켜져 있을 때 동작한다.
- 서버 저장 파일은 `.data/conflict-state.json`이며 `.gitignore`에 포함되어 있다.

## 분석 기능 상태

- 관리자 분석 카드는 현재 단순화되어 있다.
  - `Padlet학생글 엑셀 파일` 파일 선택
  - `파일 분석` 버튼
  - 결과 표시 영역
- 서버 없이도 `index.html`에서 `Padlet학생글.xlsx`를 직접 선택해 분석할 수 있게 했다.
- 브라우저에서 SheetJS CDN을 사용한다.
  - `<script src="https://cdn.jsdelivr.net/npm/xlsx@0.18.5/dist/xlsx.full.min.js"></script>`
- 분석 방식:
  - `게시물` 시트를 우선 사용
  - `본문` 컬럼을 읽음
  - 키워드 기반으로 갈등 유형 분류
  - 유형별 순위, 건수, 비율, 예시를 표시
- 서버 없이 자동으로 같은 폴더의 엑셀 파일을 몰래 읽는 것은 브라우저 보안상 불가능하므로 사용자가 파일을 직접 선택해야 한다.

## OpenAI API 상태

- Gemini는 사용하지 않도록 변경했다.
- `server.js`는 `openaiapi.env` 또는 `.env`에서 `OPENAI_API_KEY`를 읽을 수 있다.
- `openaiapi.env`는 `.gitignore`에 포함되어 있고 정적 제공 차단 대상이다.
- 기본 OpenAI 모델은 `gpt-5-mini`.
- 현재 관리자 화면의 엑셀 파일 분석은 서버 없이 로컬 브라우저 키워드 분석으로 동작하므로 OpenAI 키가 없어도 된다.

## 보안/민감정보 주의

- `openaiapi.env` 내용은 절대 읽거나 출력하지 않는다.
- `.env`, `제미나이.env.txt`, `.data/`도 브라우저로 제공하지 않도록 유지한다.
- Git 작업 전에는 `git status`로 실제 키와 학생 민감 자료가 포함되지 않았는지 확인한다.
- `Padlet학생글.xlsx`는 학생 글이 들어 있을 수 있으므로 외부 업로드/출력/로그를 피한다.

## 최근 UI/동작 변경

- 관리자 암호: `1`
- 관리자 분석 메뉴는 파일 선택/파일 분석/결과 영역만 남겼다.
- 학생 감정 삭제 영역은 상황을 직접 선택해 삭제할 수 있게 했다.
- 상황2, 상황3이 추가되어도 삭제 영역에서 원하는 상황을 고르면 해당 상황의 학생 기록만 삭제된다.
- 학생 감정 입력 삭제 후에는 localStorage 저장, 서버 동기화 시도, 워드클라우드, 학생 결과 목록이 갱신된다.
- 교사가 관리자 페이지에서 문제상황/패들렛 주소를 저장하면 `/api/class-settings`에 학생 입력과 분리해서 저장된다.
- 학생이 복사한 `index.html`을 열면 `teacherServerUrl`의 수업 설정을 자동으로 읽고, 학생 번호 선택 후에는 10초마다 문제상황/패들렛 주소를 다시 불러온다.
- 학생 복사본 동기화는 교사 컴퓨터에서 `npm start` 서버가 켜져 있고 학생 컴퓨터가 같은 네트워크에서 `http://192.168.1.19:3000`에 접속할 수 있을 때 동작한다. 교사 컴퓨터 IP가 바뀌면 `index.html`의 `teacherServerUrl`도 바꿔야 한다.
- 현재 연결 문제의 반복 원인은 사회정서앱 서버가 실제로 켜지지 않았거나, `C:\padlet-emotion-tagger\run_tagger.bat`처럼 다른 도구 배치파일을 실행한 경우다.
- `run_server.bat`은 현재 `python_social_server.py` 보조 서버를 바로 실행한다. 한글/공백 경로에서 Windows 배치가 깨지지 않도록 배치파일 문구는 ASCII로 유지한다.
- 학생 접속이 안 되면 먼저 교사 컴퓨터에서 `http://localhost:3000/api/class-settings`가 열리는지 확인한다. 이것도 안 열리면 서버가 꺼진 것이다.

## 검증 상황

- 여러 차례 `git diff --check` 통과.
- 현재 작업 환경의 셸에서는 `node`/`npm`이 PATH에 잡히지 않아 `npm start` 실행 검증은 못 했다.
- 서버 없이 엑셀 분석은 최신 Chrome/Edge에서 SheetJS CDN이 로드되어야 안정적으로 동작한다.

## 다음 작업 시 우선 확인

1. 사용자가 `사회정서앱 읽어`라고 하면 이 파일과 `AGENTS.md`를 읽는다.
2. `git status --short`로 현재 변경 상태를 확인한다.
3. `index.html`에서 요청 흐름이 학생/관리자/분석/워드클라우드/삭제 중 어디인지 구분한다.
4. 민감 파일은 열지 않는다.
