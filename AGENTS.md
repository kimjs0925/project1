# 갈등 해결 활동 앱 에이전트 지침

이 프로젝트는 학생/관리자용 갈등 해결 활동 앱입니다. 프론트엔드는 단일 `index.html`, 백엔드는 `server.js`의 Express 서버와 OpenAI 분석/상태 동기화 프록시로 구성됩니다.

## 새 채팅 인수인계

- 사용자가 `사회정서앱 읽어`라고 말하면 먼저 `사회정서앱.md`를 읽고 현재 구현 상태, 보안 주의사항, 최근 변경 내용을 파악한 뒤 작업을 시작합니다.
- `사회정서앱.md`는 작업 인수인계 문서이며 API 키나 실제 비밀값을 포함하지 않습니다.

## 작업 원칙

1. 코딩 전에 불확실한 점을 드러냅니다.
   - 학생 화면, 관리자 화면, OpenAI/로컬 엑셀 분석, 패들렛 불러오기 중 어느 흐름을 바꾸는지 먼저 구분합니다.
   - 요구가 모호하면 조용히 구현 방향을 고르지 말고 가정과 선택지를 짧게 밝힙니다.
   - 데이터 삭제, 학생 응답 구조 변경, API 키/환경변수 처리 변경은 특히 확인합니다.

2. 단순함을 우선합니다.
   - 이 앱은 빌드 시스템 없는 바닐라 HTML/CSS/JS와 Express 앱입니다. 요청이 없으면 React, 번들러, TypeScript, 새 상태관리 라이브러리를 추가하지 않습니다.
   - 한 번만 쓰는 UI/로직에 큰 추상화를 만들지 않습니다.
   - 기존 인라인 스타일과 함수 기반 구조를 존중합니다.

3. 외과적으로 변경합니다.
   - 요청과 직접 관련된 줄만 바꿉니다.
   - 주변 CSS, 한국어 문구, 기존 함수 이름, 포맷을 덤으로 정리하지 않습니다.
   - 기존 `localStorage` 키인 `conflictAppState`와 저장 데이터 shape를 바꿀 때는 마이그레이션을 같이 고려합니다.
   - `.env`, `openaiapi.env`, `제미나이.env.txt`, API 키 값은 읽거나 노출하지 않습니다.

4. 목표 중심으로 실행합니다.
   - 버그 수정은 가능하면 먼저 재현 조건을 말하고, 수정 후 확인 방법을 남깁니다.
   - 기능 추가는 “어떤 사용자 흐름이 성공하면 완료인가”를 기준으로 작게 구현합니다.
   - 검증은 최소한 관련 흐름의 수동 확인 또는 가능한 명령 실행으로 닫습니다.

## 프로젝트 구조

- `index.html`: 학생/관리자 UI, localStorage 상태, 워드클라우드, 서버 없는 엑셀 파일 분석
- `server.js`: 정적 파일 제공, `/api/state`, `/api/analyze`, `/api/padlet`, OpenAI API 키 로딩
- `사회정서앱.md`: 새 채팅 인수인계용 현재 상태 요약
- `README.md`: 실행 방법과 기능 설명
- `.env.example`: 환경변수 예시
- `andrej-karpathy-skills/`: 참고용 원본/번역 자료. 앱 런타임의 일부가 아닙니다.

## GitHub 원격 저장소

- `origin`: `https://github.com/kimjs0925/project1.git`
- 사용자가 나중에 푸시를 요청하면 이 remote로 커밋/푸시를 준비합니다.
- 푸시 전에는 `git status`로 포함 파일을 확인하고, `.env`, `제미나이.env.txt`, 실제 API 키, 민감한 학생 자료가 staged 되지 않았는지 점검합니다.

## 실행과 검증

- 의존성 설치: `npm install`
- 서버 실행: `npm start`
- 기본 접속: `http://localhost:3000`

변경 후 가능한 확인:

- HTML/CSS/클라이언트 JS 변경: 브라우저에서 학생/관리자 탭, 저장/입력/워드클라우드 흐름 확인
- 서버 변경: `npm start`로 서버 부팅 확인
- OpenAI/패들렛 변경: API 키나 외부 네트워크가 필요한 경우, 실행 가능 여부와 한계를 명확히 보고

## 앱별 주의사항

- UI 문구는 기본적으로 한국어를 유지합니다.
- 학생 입력, 감정/공감 관련 데이터는 교육 활동 맥락의 민감한 내용일 수 있으므로 불필요한 로그나 저장을 추가하지 않습니다.
- 관리자 암호, 외부 링크, 상황/등장인물/기준 저장 흐름은 기존 사용자가 잃지 않도록 보수적으로 다룹니다.
- `server.js`는 `.env`, `openaiapi.env`, `제미나이.env.txt`, `.data/`를 정적 제공에서 차단합니다. 보안 관련 보호를 약화하지 않습니다.
- OpenAI 응답은 실패하거나 JSON이 아닌 텍스트를 반환할 수 있습니다. 사용자에게 보이는 실패 메시지는 이해하기 쉽게 유지합니다.
