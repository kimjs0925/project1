# 아침대화 보안/개인정보 체크리스트

학생 데이터, 음성, 카메라, AI API를 건드리는 변경 전에 확인한다.

## 데이터 분류

| 데이터 | 민감도 | 현재 처리 |
|---|---|---|
| 학생 번호와 비밀번호 | 높음 | `morning-shared.js`에서 학급용 코드로 확인 |
| 학생 답변과 대화 전문 | 높음 | 서버 JSON 저장, 기본 2일 보존 |
| 표정 신호 | 높음 | 카메라 화면 저장 없이 라벨/신뢰도만 기록 |
| 카메라 영상 | 매우 높음 | 브라우저에서만 사용, 저장하지 않음 |
| 원본 음성 | 매우 높음 | 브라우저 음성 인식에만 사용, 저장하지 않음 |
| AI API 키 | 매우 높음 | 서버 env 파일/환경변수에서만 읽음 |

## 변경 전 질문

- 이 변경이 학생 원문, 표정 신호, 음성, 사진, API 키 중 하나를 새 위치에 저장하는가?
- 외부 API로 보내는 데이터 범위가 늘어나는가?
- 보존 기간, 삭제 동작, 관리자 조회 범위가 바뀌는가?
- 학생에게 보이는 문장이 마음 상태를 단정하거나 진단처럼 들릴 수 있는가?

## 반드시 유지할 것

- API 키는 `server.js`에서만 사용하고 브라우저 파일에 넣지 않는다.
- 카메라 영상과 원본 음성은 저장하지 않는다.
- 표정 분석 문구는 "보일 수 있어", "네 이야기를 먼저 들을게"처럼 비단정 표현을 유지한다.
- 외부 AI가 실패해도 기본 문장으로 수업 흐름을 유지한다.
- 새 데이터 필드를 추가하면 삭제와 보존 기간을 함께 고려한다.
